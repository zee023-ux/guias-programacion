1/¿Cuáles son las cuatro características básicas de la programación orientada a objetos? Describe brevemente cada una
Respuesta
Las cuatro características básicas de la programación orientada a objetos son la abstracción, la encapsulación, la herencia y el polimorfismo. La abstracción consiste en modelar entidades del mundo real quedándose solo con la información relevante. La encapsulación permite ocultar los detalles internos de un objeto y acceder a ellos únicamente mediante métodos. La herencia permite que una clase reutilice atributos y métodos de otra clase. El polimorfismo permite que un mismo método tenga distintos comportamientos según el objeto que lo ejecute.

3/Los paradigmas anteriores a la POO, ¿qué es la programación estructurada? y ¿qué es la programación modular?
Respuesta
La programación estructurada es un paradigma que organiza el código mediante estructuras básicas como secuencia, selección y repetición, evitando saltos incontrolados. La programación modular consiste en dividir un programa en módulos o funciones independientes para facilitar la reutilización, el mantenimiento y la comprensión del código.


4/¿Qué tres elementos definen a un objeto en programación orientada a objetos?
Respuesta
Un objeto se define por su estado, su comportamiento y su identidad. El estado corresponde a sus atributos, el comportamiento a los métodos que puede ejecutar y la identidad lo diferencia de otros objetos aunque tengan los mismos valores.


5/¿Qué es una clase? ¿Es lo mismo que un objeto? ¿Qué es una instancia? ¿Todos los lenguajes orientados a objetos manejan el concepto de clase?
Respuesta
Una clase es una plantilla que define atributos y métodos. Un objeto no es lo mismo que una clase, ya que es una entidad concreta creada a partir de ella. Una instancia es un objeto creado desde una clase. No todos los lenguajes orientados a objetos utilizan el concepto de clase, como ocurre con JavaScript, que se basa en prototipos.


6/¿Dónde se almacenan en memoria los objetos? ¿Es igual en todos los lenguajes? ¿Qué es la recolección de basura?
Respuesta
Los objetos suelen almacenarse en el heap de memoria. Esto no es exactamente igual en todos los lenguajes ni en todas las implementaciones. La recolección de basura es un mecanismo automático que libera la memoria ocupada por objetos que ya no se utilizan.


7/¿Qué es un método? ¿Qué es la sobrecarga de métodos?
Respuesta
Un método es una función dentro de una clase que define acciones o comportamientos que un objeto puede realizar.
La sobrecarga de métodos ocurre cuando hay varios métodos con el mismo nombre pero diferentes parámetros (tipo o cantidad), para que un mismo concepto pueda funcionar de distintas formas según los datos que reciba.

10/Intenta ejecutar un poco de Java de forma básica, con los comandos javac y java. ¿Cómo podemos compilar el programa y ejecutarlo desde linea de comandos? ¿Java es compilado? ¿Qué es la máquina virtual? ¿Qué es el byte-code y los ficheros .class?
Respuesta
Para ejecutar un programa Java desde línea de comandos se siguen estos pasos:

1.Escribir el código fuente Java. Se crea un archivo con extensión .java. Por ejemplo:
public class MiPrograma {
public static void main(String[] args) {
System.out.println("¡Hola, mundo!");
}
}

2.Compilar el programa. Abrimos la terminal en la carpeta donde está el archivo y ejecutamos:
javac MiPrograma.java
Esto genera un archivo MiPrograma.class que contiene el bytecode, un código intermedio independiente de la plataforma.

3.Ejecutar el programa. Para ejecutar el programa usamos:
java MiPrograma
Esto ejecutará el bytecode en la JVM y mostrará en consola: ¡Hola, mundo!

Java es compilado: Sí, Java se compila, pero no directamente a código máquina. Se compila a bytecode, que luego es interpretado por la JVM en tiempo de ejecución.

.Máquina Virtual de Java (JVM): La JVM es un software que interpreta el bytecode y lo ejecuta en el sistema operativo y hardware donde corra. Esto hace que Java sea portátil, porque el mismo bytecode puede ejecutarse en cualquier plataforma con JVM. La JVM también gestiona la memoria, la recolección de basura y otros recursos del programa.

.Bytecode y archivos .class: El bytecode es un código intermedio independiente de la plataforma. Cada clase compilada genera un archivo .class que contiene el bytecode. La JVM lee este bytecode y lo ejecuta en la máquina donde corre el programa.

13/Añade ahora otro nuevo método que se llame distanciaA, que reciba un Punto como parámetro y calcule la distancia entre this y el punto proporcionado 
Respuesta
double distanciaA(Punto otro) {
    double dx = this.x - otro.x;
    double dy = this.y - otro.y;
    return Math.sqrt(dx * dx + dy * dy);
}

14/El paso del Punto como parámetro a un método, es por copia o por referencia, es decir, si se cambia el valor de algún atributo del punto pasado como parámetro, dichos cambios afectan al objeto fuera del método? ¿Qué ocurre si en vez de un Punto, se recibiese un entero (int) y dicho entero se modificase dentro de la función? 
Respuesta
En Java, cuando pasamos un objeto como Punto a un método, lo que se pasa es una copia de la referencia al objeto, no el objeto completo. Esto significa que si modificamos los atributos del objeto dentro del método, los cambios afectarán al objeto original fuera del método, porque ambos (fuera y dentro del método) apuntan a la misma instancia en memoria.
 Ejemplo con Punto:

public static void modificarPunto(Punto p) {
    p.x = 10;
    p.y = 20;
}

public static void main(String[] args) {
    Punto punto = new Punto(1, 2);
    modificarPunto(punto);
    System.out.println(punto.x + ", " + punto.y); // Imprime: 10, 20
}
En cambio, si pasamos un tipo primitivo como int, se pasa una copia del valor (paso por valor). Si modificamos ese entero dentro del método, no afectará al valor original fuera del método.

Ejemplo con int:

public static void modificarEntero(int n) {
    n = 100;
}

public static void main(String[] args) {
    int numero = 5;
    modificarEntero(numero);
    System.out.println(numero); // Imprime: 5
}
 Resumen:

Objetos → paso de referencia por valor → cambios en atributos afectan al objeto original.
Primitivos → paso por valor → cambios no afectan al valor original.

15/¿Qué es el método toString() en Java? ¿Existe en otros lenguajes? Pon un ejemplo de toString() en la clase Punto en Java
Respuesta
toString() es un método de la clase Object que devuelve una representación en forma de cadena de texto de un objeto. Por defecto, devuelve algo como Clase@hashcode, pero normalmente se sobrescribe para mostrar información más útil sobre el objeto.
Sí, existe un concepto similar en otros lenguajes orientados a objetos:
En Python se usa __str__() o __repr__().
En C# se usa ToString().

Ejemplo en la clase Punto:
public class Punto {
    private int x;
    private int y;

    public Punto(int x, int y) {
        this.x = x;
        this.y = y;
    }

    @Override
    public String toString() {
        return "Punto{x=" + x + ", y=" + y + "}";
    }

    public static void main(String[] args) {
        Punto punto = new Punto(3, 4);
        System.out.println(punto); // Imprime: Punto{x=3, y=4}
    }
}

17/Quitemos un poco de magia a todo esto: ¿Como se podría “emular”, con struct en C, la clase Punto, con su función para calcular la distancia al origen? ¿Qué ha pasado con this? 
Respuesta
Se define un struct con los atributos x e y y se crean funciones externas que operen sobre él. No hay this, porque en C se pasa el objeto explícitamente como parámetro.

#include <stdio.h>
#include <math.h>

typedef struct {
    double x;
    double y;
} Punto;

Punto crearPunto(double x, double y) {
    Punto p;
    p.x = x;
    p.y = y;
    return p;
}

double distanciaAlOrigen(Punto p) {
    return sqrt(p.x * p.x + p.y * p.y);
}

int main() {
    Punto punto1 = crearPunto(3, 4);
    double distancia = distanciaAlOrigen(punto1);
    printf("La distancia al origen es: %.2f\n", distancia);
    return 0;
}

Aquí el struct y las funciones emulan los métodos de una clase. La referencia a this no existe porque cada función recibe el struct explícitamente.
